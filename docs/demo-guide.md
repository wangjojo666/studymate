# StudyMate 5 分钟答辩演示脚本

本文档用于课程设计答辩现场演示。建议提前启动后端和前端，并准备一份较小的 TXT、PDF 或 DOCX 课程资料，避免现场等待大文件解析。

## 演示目标

用 5 分钟展示 StudyMate 的完整学习辅助闭环：

登录 → 新建课程 → 上传资料 → 等待入库 → 提问 → 查看来源 → 生成提纲 → 生成练习 → 查看学习画像 → 导出报告。

## 演示前准备

1. 启动后端：

   ```powershell
   cd D:\sunny\studymate\backend
   python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
   ```

2. 启动前端：

   ```powershell
   cd D:\sunny\studymate\frontend
   npm run dev
   ```

3. 使用默认演示账号登录：

   ```text
   demo@studymate.local / studymate-demo
   ```

4. 准备一份小型课程资料，例如 C++ 多态、函数、类与对象相关内容。

## 5 分钟演示流程

### 第 0-1 分钟：登录与课程管理

打开前端页面，使用默认账号登录。进入课程列表后，新建一门课程，例如“C++ 程序设计复习”。说明本项目采用前后端分离架构，课程数据按登录用户隔离保存。

### 第 1-2 分钟：上传资料并等待入库

进入课程详情页，上传准备好的 TXT/PDF/DOCX 资料。观察资料状态从“已上传/解析中”变成“已入库”。说明后端会在后台完成文档解析、文本切片、检索索引和知识点同步。

### 第 2-3 分钟：资料问答与来源追踪

在问答区输入问题，例如“这份资料的核心知识点是什么？”。展示回答内容，并重点指出下方来源：文件名、页码、片段预览、`retrieval_provider` 和 `llm_provider`。说明系统不会只展示答案，还能说明答案依据来自哪份资料。

### 第 3-4 分钟：生成提纲和练习

切换到“复习提纲”，点击生成。再切换到“专项练习”，生成 5 道基础题。说明默认 mock/offline 模式也能跑通课程设计演示；如果配置真实模型，可以得到更自然的生成结果。

### 第 4-5 分钟：学习画像与报告导出

进入学习诊断页面，记录一条练习结果，查看知识点掌握度、薄弱点和复习建议。最后导出 PDF 学习报告。说明学习画像采用规则公式，适合课程设计答辩解释。

## 示例提问

以下问题适合现场演示：

1. 这份资料的核心知识点是什么？
2. 请总结第 1 页内容。
3. 这门课有哪些容易混淆的概念？
4. 根据资料生成 5 道基础题。
5. 根据资料生成 3 道考试题。
6. 这门课有哪些薄弱知识点？
7. 请给我一个今天的复习计划。
8. 资料中提到的 C++ 多态是什么意思？
9. 虚函数为什么能实现运行时多态？
10. 分析这段 C++ 代码的考点。

## C++ 助教演示代码

可以粘贴下面这段代码，演示编译诊断和考点识别：

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void show() { cout << "Base"; }
};

class Derived : public Base {
public:
    void show() override { cout << "Derived"; }
};

int main() {
    Base* p = new Derived();
    p->show();
    delete p;
    return 0;
}
```

默认 `CPP_RUN_ENABLED=false`，系统处于安全演示模式，不会执行本地编译运行，但仍能展示规则考点识别和离线解释。如果需要现场演示真实 g++ 编译运行，需要在可信本机环境手动设置 `CPP_RUN_ENABLED=true`，并说明这不是完整沙箱。

## 答辩说明要点

- 项目不是商业级平台，而是课程资料智能学习辅助原型系统。
- 重点展示前后端分离、数据库建模、资料解析、RAG 基础流程和学习诊断。
- 默认 mock/hash 配置用于稳定演示，真实模型可以按 `.env.example` 切换。
- RAG 证据不足时会返回低置信拒答，不会硬编答案。
- 学习画像是规则诊断，会展示公式、最近练习摘要、来源页码和证据片段。
- SQLite 适合本地演示，后续可升级为 PostgreSQL。
